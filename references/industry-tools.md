# 药厂排班工具行业调研

> 2026-06-27 调研。用于设计参考，不是功能对标。

## QC 实验室专项排班

### Smart-QC (smart-qc.com)
- **定位**：QC 实验室资源规划+排班，全球 100+ 实验室部署，TOP15 药企中 9 家在用
- **核心能力**：输入检验项目+人员+仪器 → 自动分配任务（proprietary scheduling algorithm）
- **规模**：支持 10-400 人 QC 部门
- **产品线**：Smart-QC（QC排班）、Smart-QA（QA排班）、Smart-QD（质量仪表盘）、FDAware（合规）
- **值得借鉴**：自动排班算法逻辑、大型 QC 部门的资源建模方式

### Binocs / Bluecrux (bluecrux.com/binocs)
- **定位**：药企 QC 实验室产能规划+排班调度，也覆盖细胞基因治疗(CGT)
- **核心能力**：人员、设备、洁净室统一排期；零代码对接 LIMS/ERP
- **Gartner 推荐**：Cool Vendor
- **值得借鉴**："资源池"模型 — 把人员/设备/洁净室统一抽象为可调度资源

### AmpleLogic (amplelogic.com)
- **定位**：AI 驱动药企合规平台，含 QC 规划调度模块
- **核心能力**：LIMS+eQMS+MES+DMS 一体化；低代码构建 QC 工作流；APQR/CPV 自动生成
- **值得借鉴**：多系统一体化思路，QC 排班不是孤立的

## 仪器/设备排期

### Blue Mountain RAM (bluemountainram.com)
- **定位**：药企 QC 仪器校准+维护+排期管理
- **核心能力**：校准周期自动提醒、IQ/OQ/PQ 管理、超差处理、设备生命周期
- **值得借鉴**：仪器排期的数据模型 — 校准/维护/使用三层排期

### LabWare LIMS (labware.com)
- **定位**：全球 30,000+ 实验室的 LIMS
- **核心能力**：样品→检验→仪器→结果 完整链路；仪器调度+校准追踪+稳定性考察
- **值得借鉴**：数据模型（Sample→Test→Instrument→Result）

## 生产排期（车间）

### Siemens Opcenter (sw.siemens.com/opcenter)
- **定位**：MES + 高级排程(APS)，制药行业金标准
- **核心能力**：约束排程（设备清洗换批、洁净区切换）、批记录联动、多工厂协同
- **值得借鉴**：约束排程逻辑 — 换批清洗时间、洁净区等级切换

### Frepple (frepple.com) ⭐ 开源
- **定位**：开源 APS 排程引擎，Python 实现
- **核心能力**：需求→制造订单→资源→工序计划；甘特图可视化；REST API
- **GitHub**: github.com/frepple/frepple
- **值得借鉴**：**最值得研究的开源项目**。数据模型和排程算法可直接参考

## 多部门协同

### MasterControl (mastercontrol.com)
- **定位**：云端药企质量+生产+文档管理一体化
- **值得借鉴**：多部门串联（质量、生产、培训、供应商）的设计模式

## 三个核心设计模式

从以上工具提炼：

### 1. 资源池模型（Smart-QC/Binocs）
```
资源池 = {人员池, 仪器池, 房间池}
每个任务 = 需要 X 个资源 × Y 小时
排班 = 把任务塞进资源池的时间槽，自动避免冲突
```
→ 当前 pharma-schedule 的下一步演进方向

### 2. 约束驱动排程（Frepple/Siemens）
```
硬约束：仪器同一时间只能一人用、校准过期不能用
软约束：人员偏好、缓冲时间、优先级
排程 = 在约束下找最优解
```

### 3. 依赖链传递（MasterControl）
```
采购完成 → 触发 QC 来料检验任务
QC 放行 → 触发车间投料任务
车间完工 → 触发 QC 成品检验任务
上游延迟 → 下游自动重排
```
→ pharma-schedule 已实现此模式

## 中国市场补充

- **鼎捷/用友U8/金蝶**：国产 ERP 有生产排程模块，但 QC 排班能力弱
- **国产 LIMS**（如 珠海欧美、上海科华）：有仪器管理但排班自动化程度低
- **机会**：国内药厂 QC 排班普遍用 Excel，自动化空间大
